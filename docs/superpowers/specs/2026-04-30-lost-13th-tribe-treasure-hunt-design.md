# 잃어버린 13지파의 후예 — 가족 보물찾기 게임 기획서

- **작성일**: 2026-04-30
- **대상**: 교회 가족 수련회 (8가족, 총 24~32명)
- **소요 시간**: 30~60분 (개회 5분 / 1막 30~40분 / 클라이맥스 5~10분)
- **장소**: 펜션 실내 + 야외 운동장
- **준비물**: QR 삽입형 물리적 보물 60개, 참가자 스마트폰
- **인프라**: Vercel (Next.js 16 + Upstash Redis via Vercel Marketplace)

---

## 1. 개요

8가족이 "잃어버린 13번째 이스라엘 지파의 후예"라는 정체성을 부여받고, 펜션 곳곳에 숨겨진 가문별 보물 7개씩을 찾아 13지파의 음해자(빌런)를 추리해내는 협동·경쟁 미스터리 게임이다. 보물을 스캔할 때마다 모바일 페이지에서 13지파 시조의 일기 한 페이지가 공개되고, 정답 좌표(에스더 3:6)의 초성 한 글자가 랜덤 위치에 채워진다. 7글자 초성과 일기장 단서를 종합해 가장 먼저 음해자 "**하만**"을 입력한 가족이 13지파의 진정한 상속자로 호명되며, 진짜 보물상자를 개봉한다.

## 2. 스토리 / 세계관

### 2.1 프롤로그 (개회식에서 MC 낭독, 약 2분)

> 야곱에게는 열두 아들이 있었다. 우리는 그렇게 알고 있다.
>
> 그러나 성경 어디에도, 역사 어디에도 적히지 않은 비밀이 있다.
>
> 야곱에게는 한 명의 아들이 더 있었다. 어둠 속에서 태어나 빛 가운데 살았던 아이, **므에덴**. 그가 13번째 지파의 시조다.
>
> 13지파는 야훼의 말씀을 가장 충실히 지키던 지파였다. 그러나 가나안에 정착한 후, 그들을 시기하고 두려워한 자가 있었다. 그는 13지파의 모든 기록을 불태웠다. 성경에서, 역사에서, 사람들의 기억에서 13지파의 이름을 지웠다. 그래서 오늘날 우리는 12지파만 알고 있다.
>
> 그러나 13지파는 사라지지 않았다. 그들은 이름을 숨기고 여덟 가문으로 흩어져, 후손들에게 단 하나의 사명을 남겼다.
>
> *"누가 우리를 지웠는지 찾아라. 그날, 13지파의 이름은 빛으로 돌아온다."*
>
> 100세대가 흐른 오늘, 여덟 가문이 한 자리에 다시 모였다. 너희가 그 후예다. 너희가 보물을 찾을 때마다 음해자의 흔적이 한 글자씩 드러날 것이다. 가장 먼저 음해자를 지목하는 가문이, 13지파의 진짜 상속자가 된다.

### 2.2 정답 / 빌런

- **음해자**: **하만** (Haman)
- **정답 좌표**: 에스더 3장 6절 — *"하만이 모르드개의 민족을 다 멸하고자 하더라"*
- **픽션 설정**: 하만이 죽기 직전 마지막 저주를 부려 페르시아 왕궁 서고를 불태웠고, 거기 있던 13지파의 모든 기록이 함께 사라졌다. 12지파는 에스더의 헌신으로 살아남았으나 13지파의 이름만은 그날 이후 누구도 입에 담지 않게 되었다.
- **선정 근거**: 성경 통틀어 한 민족 자체를 역사에서 지우려 한 유일한 인물 + 한국 교회 가족 인지도 최상 + 권선징악 카타르시스 강함.

### 2.3 13지파 시조의 일기 (7페이지, 모든 가족 공유)

가족이 보물을 찾을 때마다 1→2→…→7 순서로 자동 공개된다. 어느 보물을 먼저 스캔하든 일기는 항상 다음 페이지가 열린다.

**Page 1 — 시작**
> 야곱의 열셋째 아들, 므에덴이 태어난 밤이었다. 어머니는 그를 품에 안고 말했다. *"에덴에서 태어난 아이여, 너는 내 마지막 약속이다."* 그날부터 13지파의 빛이 시작되었다.

**Page 2 — 번성**
> 13지파는 광야의 천막에서 가장 신실한 등불을 켰다. 야훼는 그들에게 별과 들풀, 옛글과 도구, 노래와 빛깔과 바람을 가르쳤다. 그들은 이름을 자랑하지 않았기에, 형제들 사이에서 가장 깊이 사랑받는 자들이라 불렸다.

**Page 3 — 약속의 땅**
> 가나안에 들어선 후, 13지파는 자신의 땅을 받지 않았다. 대신 흩어져 12지파 모두의 곁에 머물렀다. 누구의 그림자도 되지 않고, 누구의 빛이 될 수 있도록.

**Page 4 — 시기의 시작**
> 그러나 시기하는 자가 있었다. 그는 우리의 빛을 두려워했다. 자신이 가지지 못한 것을 가진 자가 있다는 사실 자체를 견딜 수 없었다. 그는 우리의 이름을 지우기로 결심했다.

**Page 5 — 첫 흔적**
> 처음에는 한 권의 두루마리가 사라졌다. 다음에는 한 마을이 사라졌고, 그 다음에는 한 가문 전체가 기억에서 사라졌다. 우리의 이름을 입에 담는 자는 모두 화를 당했다.

**Page 6 — 마지막 결단**
> 장로들이 모였다. *"우리는 사라질 것이다. 그러나 흔적은 남길 것이다."* 그들은 여덟 가문에게 여덟 가지 도구를 나눠주었다. 별과 지도, 들풀과 옛글, 도구와 노래, 빛깔과 바람. *"후손들이 이 모든 것을 모으는 날, 음해자의 이름이 드러난다."*

**Page 7 — 후손에게 (마지막 페이지)**
> 내 이름조차 너희에게 전할 수 없다. 그들은 내 이름까지도 지워버렸을 테니. 그러나 너희가 이 글을 읽고 있다면, 우리는 사라지지 않은 것이다. 음해자의 이름을 부르라. 그날, 13지파의 이름이 다시 빛으로 돌아온다. — *13지파 마지막 장로*

### 2.4 클라이맥스 스크립트 (우승 가족 호명 후, MC 낭독)

> ○○○ 가족이 음해자의 이름을 불렀다. 그의 이름은 — **하만**. 100세대 동안 잠겨 있던 13지파의 이름이 오늘 다시 빛 가운데 부른다. 우리가 여기 있다.
>

## 3. 게임 메커니즘

### 3.1 전체 흐름

```
[T-0: 개회식 5분]
  ├ MC 프롤로그 낭독 (또는 mp3 재생)
  ├ 8가족에게 게임 URL + 첫 QR 진입 카드 배포
  └ 모바일 페이지 접속 → 가족 닉네임 등록 → 가문 색상 자동 배정

[1막: 보물 수집 + 추리 30~40분]
  ├ 가족이 자기 가문 보물 7개를 찾으러 흩어짐
  ├ 보물 QR 스캔 시 모바일에서:
  │     ① 일기장 다음 페이지 공개 + 🔊 mp3 재생 가능
  │     ② 7칸 초성판의 빈 점 1곳에 랜덤 배치된 글자 등장
  │     ③ 진행도 표시 (예: 4/7)
  │     ④ 정해진 위치(4번째, 7번째 스캔)에서 스페셜 힌트 카드 등장
  ├ 가족은 언제든 "정답 입력" 버튼으로 추리 시도 가능
  │     - 시도 횟수 = 보물 찾은 개수
  │     - 정답: "하만" (대소문자/영어 표기 허용)
  └ 첫 정답 발생 시 → 모든 가족 모바일에 클라이맥스 호출 알림

[클라이맥스 5~10분]
  ├ 8가족이 거실 큰 테이블에 모임
  ├ 우승 가족 호명 + 클라이맥스 mp3 낭독
  ├ 다 같이 에스더 3:6 낭독
  ├ 우승 가족이 진짜 보물상자 개봉
  └ 가족별 선물 분배 + 단체사진
```

### 3.2 닉네임 등록 (로그인)

- 모바일 페이지 첫 접속 시 닉네임 입력 화면 (예: "김씨네", "박OO 가족")
- **신규 닉네임** 등록 시:
  - 가문 색상 8개(빨강·주황·노랑·초록·파랑·남색·보라·갈색) 중 미사용 색 자동 배정
  - `familyId` 발급 + `nickname-index:{nickname}` 키로 매핑 저장 + 클라이언트 쿠키에 저장
- **기존 닉네임** 입력 시 (같은 가족 다른 디바이스):
  - 서버가 `nickname-index`에서 기존 `familyId` 조회 → 새 디바이스도 동일 `familyId` 쿠키 발급
  - 진행 상태(찾은 보물, 일기장, 시도 횟수 등) 즉시 공유
  - 한 디바이스에서 보물 스캔 시 다른 디바이스도 폴링으로 상태 갱신 확인
- 닉네임은 게임 진행 도중 변경 불가
- 8가족 슬롯이 모두 차면 신규 닉네임 등록 차단 (운영자가 admin에서 슬롯 회수 가능)

### 3.3 보물 스캔 메커니즘

- **모든 보물은 자유 수집** — 가족 귀속 없음. 어느 가족이든 먼저 찾는 가족이 가져감.
- 각 보물에는 고유 UUID 기반 QR 코드 부착. URL 형식: `https://[domain].vercel.app/scan/{uuid}`
- 보물 60개는 동일한 풀(pool)에서 자유 경쟁. 한번 어떤 가족이 스캔(=등록)하면 그 보물은 다른 가족도 자기 자신도 다시 등록 불가.
- 보물 스캔 시 백엔드 처리:
  1. 보물 UUID 유효성 검증 (등록되지 않은 UUID는 *"알 수 없는 보물입니다"*)
  2. 이미 누군가 스캔한 보물이면:
     - 자기 가족이 이미 등록 → *"이미 등록한 보물입니다"*
     - 다른 가족이 등록 → *"이 보물은 ○○○ 가족이 이미 발견했습니다"*
  3. 신규 등록 시:
     - `family.treasuresFound.push(uuid)` + `treasure.claimedByFamilyId = familyId`
     - 일기장 다음 페이지 unlock (`diaryPagesRead = min(treasuresFound.length, 7)`)
     - 7칸 초성 배열 중 빈 위치가 있으면 1곳 랜덤 선택 → 해당 위치 글자 공개
     - 빈 위치 없음 (이미 7개 다 채워짐) → 초성 변화 없이 시도 횟수만 +1
     - 스캔 순서(treasuresFound.length)가 4 또는 7이면 → 스페셜 힌트 큐에 추가

### 3.4 7칸 초성 배열

- 정답 좌표 "에스더 3장 6절"의 한국식 초성 7글자
- 배열: `[ㅇ, ㅅ, ㄷ, ㅅ, ㅈ, ㅇ, ㅈ]`
- 단어 그루핑은 **표시하지 않음** — 7개 점이 한 줄로 나열됨 (`· · · · · · ·`)
- 보물 1개 스캔 시 빈 위치 중 랜덤 1곳에서 점이 글자로 변환
- 채워지는 순서는 가족마다 다름 (= 운이지만 결과적으로는 모두 동일하게 7글자 다 보게 됨)

**예시 진행** (한 가족 기준 — 7번 자리 = "절" → ㅈ 의 정답표 기준):

```
정답표:    ㅇ  ㅅ  ㄷ  ㅅ  ㅈ  ㅇ  ㅈ
위치 번호:  1   2   3   4   5   6   7

스캔 0:  · · · · · · ·     (시작)
스캔 1:  · · · ㅅ · · ·     (4번 위치 채워짐 → ㅅ)
스캔 2:  ㅇ · · ㅅ · · ·    (1번 위치 채워짐 → ㅇ)
스캔 3:  ㅇ · · ㅅ · ㅇ ·    (6번 위치 채워짐 → ㅇ)
스캔 4:  ㅇ ㅅ · ㅅ · ㅇ ·   (2번 위치 → ㅅ) + 🏆 스페셜 힌트 1
스캔 5:  ㅇ ㅅ · ㅅ ㅈ ㅇ ·  (5번 위치 → ㅈ)
스캔 6:  ㅇ ㅅ · ㅅ ㅈ ㅇ ㅈ  (7번 위치 → ㅈ)
스캔 7:  ㅇ ㅅ ㄷ ㅅ ㅈ ㅇ ㅈ + 🏆 스페셜 힌트 2  (3번 위치 → ㄷ)
```

> 7개를 다 모은 시점의 최종 형태는 항상 `ㅇ ㅅ ㄷ ㅅ ㅈ ㅇ ㅈ` (= 에스더 3장 6절). 채워지는 순서만 가족마다 다름.

- **트리거**: 자기 가문의 4번째 보물 스캔 / 7번째 보물 스캔
- **내용**: 모든 가족 동일

| 4번째 스캔 | *"음해자는 이스라엘 백성이 아니다. 멀리서 온 이방인이었다."* |
| 7번째 스캔 | *"그는 왕의 그림자였다. 왕보다 더 많은 백성을 두려워했다."* |

- 모바일 페이지에 별도 카드 UI로 누적 표시 (📜 아이콘)

### 3.6 정답 입력

- 모바일 페이지 하단에 "🔍 정답 입력" 버튼 상시 표시
- 클릭 시 모달:
  - 텍스트 박스 1개 (자유 입력)
  - "이건 힌트입니다 — 음해자의 이름을 적어주세요" 안내문
  - 남은 시도 횟수 표시 (예: *"3회 남음"*)
  - 제출 버튼
- **시도 횟수 산식**: `attemptsRemaining = treasuresFound - attemptsUsed`
- **정답 판정**: 입력값을 trim·소문자화·공백 제거 후 다음 집합과 매치하면 정답
  - `["하만", "haman"]`
  - 오타 허용 안 함 (단순 매치)
- **오답**: `attemptsUsed += 1` → "아직 아닙니다. 보물을 더 찾아 단서를 모으세요."
- **시도 0회**: 입력 모달 비활성화, *"보물을 더 찾으면 다시 시도할 수 있습니다"*
- **정답**: 즉시 우승 처리 → 글로벌 게임 상태에 `winnerFamilyId` 기록 → 모든 가족 모바일에 푸시(폴링) 알림

- 우승 가족 외 가족은 자기 진행 상태(찾은 보물 수, 일기장 페이지) 그대로 보존되어 회고에 활용 가능
- 8가족 모두 같은 화면을 봄 (우승 가족도, 다른 가족도)

## 5. 보물 자산 명세

### 5.1 보물 ID 체계 (UUID 기반 자유 수집)

- 총 60개 물리적 보물, 모두 동일한 풀에서 자유 경쟁
- 각 보물에 고유 UUID 부여 (예: `e7c9f2a1-4b8d-4f5e-9a3c-1d6e8f0a2b5c`)
- 가족-보물 사전 매핑 없음. 어느 가족이든 먼저 찾으면 자기 것
- 한 보물은 게임 전체에서 단 한 번만 등록 가능. 이미 등록된 보물 재스캔 시 *"이 보물은 ○○○가족이 이미 발견했습니다"* 메시지

**전략적 함의**:
- 60개 / 8가족 = 가족당 평균 7.5개 → 7개 채우기에 충분
- 빠른 가족: 7개 이상 수집 (7개로 초성 다 채워지고, 추가 수집은 시도 횟수만 +1)
- 느린 가족: 7개 미만으로 끝날 가능성 → 초성이 다 안 채워진 채 추리해야 함 → 추리 난이도 ↑

### 5.2 QR 코드 인쇄 가이드

- 보물 60개 모두에 QR 부착 (백업 개념 없음 — 모두 활성)
- 인쇄 형식: 50×50mm 스티커, 보물에 붙이기
- 모든 QR 흑백 동일 디자인
- QR이 가리키는 URL: `https://[domain].vercel.app/scan/{uuid}`
- UUID 60개는 빌드 타임에 사전 생성 → DB(`treasure:{uuid}`)에 미리 등록 → QR 생성 → 인쇄

### 5.3 보물 배치 가이드

- 펜션 실내 + 야외 운동장 전체에 골고루 분산
- 4세 어린이가 닿을 수 있는 높이도 일부 포함
- 너무 어려운 곳(높은 곳, 위험한 곳) 금지
- 펜션 ↔ 운동장 양쪽에 균등 배치 (한쪽에 몰리면 동선 정체)
- 운영자가 사전에 배치 지도를 작성해 게임 후 미발견 보물 회수 가능하도록 (어떤 UUID가 어디 있었는지 기록)

## 6. 음성 자산 (mp3)

총 10개 mp3 파일을 사전 녹음 또는 AI 보이스(예: ElevenLabs, 네이버 클로바)로 생성.

| 파일 | 용도 | 길이 (목표) | 트리거 |
|---|---|---|---|
| `prologue.mp3` | 개회식 프롤로그 (내레이션) | 90초 | 운영자 수동 / 첫 화면 자동 |
| `diary-1.mp3` ~ `diary-7.mp3` | 일기장 7페이지 (내레이션) | 각 30~60초 | 모바일 🔊 버튼 |
| `climax.mp3` | 우승 호명 + 클라이맥스 낭독 (내레이션) | 90초 | 우승 발생 직후 자동 재생 |
| `ending.mp3` | 게임 종료 축하 음악 (BGM) | 60~120초 | climax.mp3 종료 후 자동 이어 재생 |

- 내레이션 권장: 차분한 중년 남성 또는 여성 톤
- 효과음(불꽃, 바람, 두루마리 펼치는 소리 등) 가미 권장
- `ending.mp3`은 보컬 없는 웅장하고 따뜻한 BGM (저작권 프리 음원 활용 권장 — 예: YouTube Audio Library, Pixabay Music)
- 파일 위치: Next.js `public/audio/*.mp3`
- 모바일 페이지 🔊 버튼 클릭 시 HTML5 `<audio>` 재생
- 게임 종료 시 클라이맥스 화면에서 climax → ending 순으로 자동 재생, 사용자 일시정지 가능

### 7.3 데이터 모델 (Vercel KV)

```typescript
// game:state (singleton)
type GameState = {
  status: "idle" | "running" | "finished"
  startedAt?: number  // unix ms
  winnerFamilyId?: string
  winnerNickname?: string
  finishedAt?: number
}

// family:{familyId}
type Family = {
  id: string                    // uuid
  nickname: string
  color: string                 // hex
  treasuresFound: string[]      // [treasureUuid, ...] in scan order
  initialsRevealed: Array<{ position: number; char: string }>
  diaryPagesRead: number        // 0~7
  attemptsUsed: number
  specialHintsSeen: number[]    // [4, 7]
  createdAt: number
}

// nickname-index:{nickname}
type NicknameIndex = string  // → familyId

// treasure:{uuid} — 빌드 타임에 60개 사전 등록
type Treasure = {
  uuid: string
  claimedByFamilyId?: string    // null = 미발견
  claimedAt?: number
}

// families:list — 전체 가족 ID 리스트 (랭킹/현황용)
type FamiliesList = string[]   // [familyId, ...]
```

### 7.4 핵심 알고리즘

**보물 스캔 (`POST /api/scan/[uuid]`)**

```
1. familyId from session/cookie
2. fetch treasure:{uuid} from KV
3. if treasure not exists:
     return { error: "알 수 없는 보물입니다" }
4. if treasure.claimedByFamilyId exists:
     if claimedByFamilyId === familyId:
       return { error: "이미 등록한 보물입니다" }
     else:
       owner = fetch family:{claimedByFamilyId}
       return { error: `이 보물은 ${owner.nickname} 가족이 이미 발견했습니다` }
5. // atomic CAS — claim treasure
   if !atomicSetIfEmpty(treasure:{uuid}, claimedByFamilyId=familyId):
     // race lost
     re-fetch treasure → return owner message
6. fetch family:{familyId}
7. // process scan
   family.treasuresFound.push(uuid)
   family.diaryPagesRead = min(family.treasuresFound.length, 7)
   // pick random unfilled position (only if positions remain)
   correctInitials = ["ㅇ","ㅅ","ㄷ","ㅅ","ㅈ","ㅇ","ㅈ"]
   filledPositions = family.initialsRevealed.map(r => r.position)
   unfilled = [0..6].filter(p => !filledPositions.includes(p))
   if unfilled.length > 0:
     randomPos = pickRandom(unfilled)
     family.initialsRevealed.push({ position: randomPos, char: correctInitials[randomPos] })
   // (else: 초성 이미 다 채워짐 — 시도 횟수만 +1 효과)
   // special hint?
   if (family.treasuresFound.length === 4 || family.treasuresFound.length === 7):
     family.specialHintsSeen.push(family.treasuresFound.length)
8. save family to KV
9. return updated family state
```

**다른 가족 진행도 조회 (`GET /api/families/progress`)**

```
1. fetch families:list
2. for each familyId, fetch family.{nickname, color, treasuresFound.length}
3. return [{ nickname, color, count, isMe }] sorted by count desc
```

**정답 입력 (`POST /api/answer`)**

```
1. fetch game.state — if status === "finished", return { tooLate: true, winner: ... }
2. fetch family
3. attemptsRemaining = family.treasuresFound.length - family.attemptsUsed
4. if attemptsRemaining <= 0:
     return { error: "시도 횟수가 없습니다" }
5. normalize input: trim, lowercase, remove spaces
6. if normalized in ["하만", "haman"]:
     // atomic CAS on game state to prevent race
     if game.state.status === "running":
       game.state = { status: "finished", winnerFamilyId, winnerNickname, finishedAt: now }
       save
       return { won: true }
     else:
       return { tooLate: true, winner: existingWinner }
7. else:
     family.attemptsUsed += 1
     save
     return { wrong: true, attemptsRemaining: ... }
```

### 7.5 폴링 / 우승 감지

- 메인 게임 화면에서 `GET /api/game/state` 3초 주기 폴링
- `status === "finished"` 감지 시 클라이맥스 화면으로 자동 전환
- 폴링 주기는 환경 변수로 조정 가능

### 7.6 인프라 운영

- **도메인**: Vercel 자동 도메인 사용 (예: `tribe-13.vercel.app`) 또는 커스텀 도메인
- **환경 변수**: `UPSTASH_REDIS_REST_URL`, `UPSTASH_REDIS_REST_TOKEN`, `ADMIN_PASSWORD`, `NEXT_PUBLIC_BASE_URL`
- **Upstash Redis 연결**: Vercel 프로젝트 → Storage 탭 → Marketplace에서 Upstash Redis 추가 → 환경 변수 자동 주입

## 12. 결정 이력 (브레인스토밍 요약)
